# ContextGate Agent Guide

This is the canonical instruction file for coding agents working in this repository. Tool-specific
files must remain thin pointers to this file.

## Product Contract

ContextGate is a production-oriented, single-tenant RAG admission gateway. Its primary value is
deciding whether an answer may be returned and preserving the evidence for that decision.

- The primary result is `answered`, `abstained`, or `blocked`.
- Abstained and blocked results must have empty answer text and no citations.
- Operational failures are typed HTTP errors, not admission decisions.
- Citation index, chunk ID, and source must resolve to the same retrieved hit.
- Verification, pricing, provider, deadline, risk, or policy uncertainty must not fail open.
- Client request IDs are correlation metadata; run IDs are generated by the server.
- Response caching remains disabled until the version-complete contract in the docs is implemented.

## Architecture

Dependencies point inward: `domain -> application -> ports -> adapters -> apps`. Domain and
application code must not import FastAPI, SQLAlchemy, Qdrant, Celery, LiteLLM, MLflow, or Chainlit.
Add a port only for a real external boundary. Compose objects in `apps/container.py`.

See `docs/repository-map.md` and `docs/architecture.md` before moving responsibilities.

## Commands

```bash
uv sync --locked --extra dev
uv run ruff check .
uv run ruff format --check .
uv run mypy src tests
uv run deptry .
uv audit --locked
uv run pytest --cov=contextgate --cov-report=term-missing
uv run alembic -c src/contextgate/adapters/sqlalchemy/alembic.ini upgrade head
uv run alembic -c src/contextgate/adapters/sqlalchemy/alembic.ini check
docker compose --env-file .env.example config --quiet
```

Use the deterministic embedding backend for unit/CI tests. Do not make tests depend on a
developer's `.env`, running services, downloaded model, clock race, or network.

## Change Rules

- Preserve public status/reason values or add an explicit migration note.
- Add focused unit tests for pure behavior and adapter contract tests for integrations.
- Apply schema changes only through a new Alembic revision; never rewrite a released revision.
- Keep API scopes explicit and update OpenAPI/docs for every route change.
- Keep uploads inside managed roots and preserve size, signature, extraction, and timeout limits.
- Never log secrets, full API keys, document bodies, prompts, or unredacted traces by accident.
- Never publish synthetic demo metrics as production benchmarks.
- Do not commit `.env`, databases, model caches, reports, MLflow data, or customer content.
- Preserve unrelated workspace changes.

## Definition Of Done

The change has tests proportional to risk, Ruff/format/MyPy/Deptry pass, migrations have no drift,
documentation examples match behavior, and the relevant clean Compose workflow passes. Security,
admission, budget, job, ledger, and policy boundaries target at least 90% module coverage.

If a release claim changes, update `README.md`, `docs/limitations.md`, `docs/launch.md`, and the
machine-readable release report together.
